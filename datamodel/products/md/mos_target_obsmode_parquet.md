## Basic Information
The table contains observing constraints for different observing modes. Cadences have an observing mode associated with each epoch, and the constraints are checked before an epoch can be scheduled.

### Naming Convention
$MOS_TARGET/[V_TARG]/[FTYPE]/mos_obsmode.[FTYPE] with FTYPE='parquet' and V_TARG='2.0.0' for DR20.

## Example DATAFRAME List

### Columns


#### label

- Column Name: label
- Description: Descriptive name for the observing mode, e.g. 'dark_rm'
- Unit: N/A
- Data Type: String


#### min_moon_sep

- Column Name: min_moon_sep
- Description: Minimum distance a target must be from the Moon
- Unit: deg
- Data Type: Float32


#### min_deltav_ks91

- Column Name: min_deltav_ks91
- Description: Minimum estimated background flux from the Moon, using the method of Krisciunas, K., & Schaefer 1991
- Unit: mag
- Data Type: Float32


#### min_twilight_ang

- Column Name: min_twilight_ang
- Description: Minimum twilight angle (angle of the Sun below the horizon) for a target to be observed
- Unit: deg
- Data Type: Float32


#### max_airmass_apo

- Column Name: max_airmass_apo
- Description: Max airmass at which a target can be observed at APO
- Unit: N/A
- Data Type: Float32


#### max_airmass_lco

- Column Name: max_airmass_lco
- Description: Max airmass at which a target can be observed at LCO
- Unit: N/A
- Data Type: Float32
