## Basic Information
Cadences define the time between series of exposures (one "epoch") on a field, and the observing conditions, e.g., skybrightness, for each epoch.

### Naming Convention
$MOS_TARGET/[V_TARG]/[FTYPE]/mos_cadence.[FTYPE] with FTYPE='parquet' and V_TARG='2.0.0' for DR20.

### Releases

## Example DATAFRAME List

### Columns


#### label

- Column Name: label
- Description: Descriptive name for the cadence with a version
- Unit: N/A
- Data Type: String


#### nepochs

- Column Name: nepochs
- Description: Number of epochs in the cadence
- Unit: N/A
- Data Type: Int32


#### pk

- Column Name: pk
- Description: Primary key
- Unit: N/A
- Data Type: Int64


#### label_root

- Column Name: label_root
- Description: Descriptive name, typically {bright/dark}_{nepochs}x{nexps}
- Unit: N/A
- Data Type: String


#### label_version

- Column Name: label_version
- Description: Version; cadences are added in versioned batches
- Unit: N/A
- Data Type: String


#### max_skybrightness

- Column Name: max_skybrightness
- Description: The maximum skybrightness for all epochs in this cadence
- Unit: N/A
- Data Type: Float32


#### nexp_total

- Column Name: nexp_total
- Description: The total number of exposures planned for this cadence
- Unit: N/A
- Data Type: Int32
