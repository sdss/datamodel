## Basic Information
Summary QA/QC FITS data product for LVM DR20 exposures, containing one row per exposure with target identifiers, a packed QA bitmask, and decoded per-metric QA flags for DRP and DAP quality assessment. The metrics summarize automated and visual checks discussed for the DR20 QA process, including sky brightness, flux calibration, wavelength calibration, flat-fielding, sky subtraction, DAP fitting quality, and visual inspection flags.

### Naming Convention
$LVM_SPECTRO_ANALYSIS/[DRPVER]/[DAPVER]/qa_bitmasks-[DRPVER]-[DAPVER].fits

### HDU1: QA
Binary table containing one row per DR20 exposure with exposure identifiers, target type, a packed QA bitmask, and decoded QA flags for the DR20 DRP and DAP quality-control metrics.

#### HDU Type: BINARY TABLE
#### HDU Size:  32 KB

##### Header Table Caption for HDU1
Key | Value | Comment | |
| --- | --- | --- | --- |
| XTENSION | BINTABLE | binary table extension |
| BITPIX | 8 | array data type |
| NAXIS | 2 | number of array dimensions |
| NAXIS1 | 195 | length of dimension 1 |
| NAXIS2 | 169 | length of dimension 2 |
| PCOUNT | 0 | number of group parameters |
| GCOUNT | 1 | number of groups |
| TFIELDS | 20 | number of table fields |
| TARG_TYP | Target types: MW=0, MCs=1, Extragalactic=2 |  |
| QA_ENC | 2-bit GOOD=00 WARNING=01 MAJOR_ISSUE=10 UNDEFINED=11 |  |
| QA_BITS | 2-bit fields packed LSB-first in this order (see DESC extension): bright stars, sky brightness, absflux, relflux br, relflux rz, wavecal b, wavecal r, wavecal z, flatfield, sky contsub, sky linesub, dap npfit, dap pmfit, visual drp, visual dap |  |
| EXTNAME | QA | extension name |

##### Binary Table Caption for HDU1
Name | Type | Unit | Description |
| --- | --- | --- | --- |
 | expnum | int64 | dimensionless | Unique exposure number identifying the observation; provided by the DRP summary table used for the DR20 QA workflow. |
 | mjd | int64 | day | Modified Julian Date of the observation. |
 | tileid | int64 | dimensionless | Identifier of the LVM tile associated with the exposure. |
 | target_type | int64 | integer code | Target-type code indicating if the exposure is of a target in the Milky Way (0), Malleganic Clouds (1), or a Local Volume galaxy (2). |
 | qa_bitmask | char[10] | packed hexadecimal bitmask | Packed QA bitmask using 2-bit fields stored least-significant-bit first in the metric order documented in the QA header and DESC extension; encoding is GOOD=00, WARNING=01, MAJOR ISSUE=10, UNDEFINED=11. |
 | qa_brightstars | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for contamination or quality impact from bright stars in the field. |
 | qa_sky_brt | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for sky brightness conditions during the exposure, related to quantities such as lunar fractional illumination, Moon altitude, Moon separation, and twilight conditions. |
 | qa_absflux | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for absolute flux-calibration quality based on comparisons with external photometric datasets (Gaia stars in the field). |
 | qa_relflux_br | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for relative flux calibration between the blue and red spectrograph arms, used to assess spectrophotometric stability and possible continuum jumps between arms. |
 | qa_relflux_rz | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for relative flux calibration between the red and infrared/z spectrograph arms, used to assess spectrophotometric stability and possible continuum jumps between arms. |
 | qa_wavecal_b | char[9] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for wavelength-calibration quality in the blue channel, assessed using prominent night-sky lines. |
 | qa_wavecal_r | char[9] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for wavelength-calibration quality in the red channel, assessed using prominent night-sky lines. |
 | qa_wavecal_z | char[9] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for wavelength-calibration quality in the infrared/z channel, assessed using prominent night-sky lines. |
 | qa_flatfield | char[9] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for flat-fielding quality, assessed using fiber-by-fiber behaviour of the prominent night-sky lines. |
 | qa_skycont_sub | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for sky-continuum subtraction quality, based on residual continuum levels after sky subtraction. |
 | qa_skyline_sub | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for sky-emission-line subtraction quality, based on residuals for selected sky lines or regions such as [OI]5577, [OH]6865, [Na]5891, and O2. |
 | qa_dap_npfit | char[9] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for the consistency between the DAP parametric and non-parametric emission-line measurements. |
 | qa_dap_pmfit | char[9] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag for the quality of DAP parametric emission-line fitting, based on the residuals after the Gaussian model subtraction for the bright emission lines. |
 | qa_visual_drp | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag from visual inspection of DRP products, including SFrame-level artifacts and data-quality issues identified during the DR20 inspection process. |
 | qa_visual_dap | char[11] | 2-bit QA flag encoded as GOOD, WARNING, MAJOR ISSUE, or UNDEFINED | Decoded QA flag from visual inspection of DAP products, including maps, fits, residuals, and other DAP-level artifacts identified during the DR20 inspection process. |



### HDU2: DESC
Binary table documenting the QA metrics encoded in the QA extension, including the bit positions, metric descriptions, and criteria for WARNING and MAJOR ISSUE flags.

#### HDU Type: BINARY TABLE
#### HDU Size:  4 KB

##### Header Table Caption for HDU2
Key | Value | Comment | |
| --- | --- | --- | --- |
| XTENSION | BINTABLE | binary table extension |
| BITPIX | 8 | array data type |
| NAXIS | 2 | number of array dimensions |
| NAXIS1 | 327 | length of dimension 1 |
| NAXIS2 | 15 | length of dimension 2 |
| PCOUNT | 0 | number of group parameters |
| GCOUNT | 1 | number of groups |
| TFIELDS | 5 | number of table fields |
| EXTNAME | DESC | extension name |

##### Binary Table Caption for HDU2
Name | Type | Unit | Description |
| --- | --- | --- | --- |
 | Metric | char[14] | dimensionless | Name of the QA metric corresponding to one decoded QA flag and one packed 2-bit field in qa_bitmask. |
 | Bits | char[5] | bit index | Bit positions assigned to the metric within the packed qa_bitmask. |
 | Description | char[67] | dimensionless | Text description of the QA metric and the data-quality aspect it summarizes. |
 | WARNING | char[105] | dimensionless | Criterion or explanatory text for assigning the WARNING state to the metric; exact quantitative thresholds may be provided in the DR20 webpages. |
 | MAJOR ISSUE | char[136] | dimensionless | Criterion or explanatory text for assigning the MAJOR ISSUE state to the metric; exact quantitative thresholds may be provided in the DR20 webpages. |
