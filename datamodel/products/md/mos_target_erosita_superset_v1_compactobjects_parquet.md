## Basic Information
One of several tables describing optical/IR counterparts to eROSITA
X-ray sources identified via various methods.

These tables contain a superset of potential targets from which the SDSS-V spectroscopic targets were drawn. The dr20_erosita_superset_v1_compactobjects table includes counterparts to point-like X-ray sources, chosen via algorithms optimised to select compact objects. Each row corresponds to one possible match between an X-ray source and a potential optical/IR counterpart. The X-ray columns (ero_*) record the eROSITA information known at the time of target selection and may differ from publicly available eROSITA catalogs. The dr20_erosita_superset_v1* tables are derived from a combination of eROSITA's first 6-month survey of of the West Galactic hemisphere ('eRASS1'), and from the eROSITA observations of the eROSITA Final Equatorial Depth performance verification field ('eFEDS').

### Naming Convention
$MOS_TARGET/[V_TARG]/[FTYPE]/mos_erosita_superset_v1_compactobjects.[FTYPE] with FTYPE='parquet' and V_TARG='2.0.0' for DR20.

## Example DATAFRAME List

### Columns


#### ero_version

- Column Name: ero_version
- Description: Identifier giving the eROSITA dataset and processing version e.g. 'eFEDS_c940', 'em01_c946_201008_poscorr' etc
- Unit: N/A
- Data Type: String


#### ero_detuid

- Column Name: ero_detuid
- Description: The standard official eROSITA unique detection identifier, e.g. 'em01_123456_020_ML12345_001_c946' etc
- Unit: N/A
- Data Type: String


#### ero_flux

- Column Name: ero_flux
- Description: X-ray flux
- Unit: erg/cm2/s
- Data Type: Float32


#### ero_flux_type

- Column Name: ero_flux_type
- Description: Flux range used to compute ero_flux, e.g. '0.2-2.3keV', '0.5-2keV', '0.5-8keV' etc
- Unit: N/A
- Data Type: String


#### ero_mjd

- Column Name: ero_mjd
- Description: This column is empty
- Unit: N/A
- Data Type: String


#### ero_morph

- Column Name: ero_morph
- Description: Morphological classification of X-ray source, e.g. 'pointlike', 'extended', 'unknown', ...
- Unit: N/A
- Data Type: String


#### ero_flags

- Column Name: ero_flags
- Description: Flags indicating properties of the X-ray detection
- Unit: N/A
- Data Type: Int64


#### ero_det_like

- Column Name: ero_det_like
- Description: Detection likelihood of X-ray source
- Unit: %
- Data Type: Float32


#### ero_ra

- Column Name: ero_ra
- Description: Best determination of X-ray position (J2000)
- Unit: deg
- Data Type: Float64


#### ero_dec

- Column Name: ero_dec
- Description: Best determination of X-ray position (J2000)
- Unit: deg
- Data Type: Float64


#### ero_radec_err

- Column Name: ero_radec_err
- Description: Best estimate of X-ray position uncertainty
- Unit: arcsec
- Data Type: Float32


#### xmatch_method

- Column Name: xmatch_method
- Description: The X-ray-to-optical/IR cross-match method that was used in this case e.g. 'XPS-ML/NWAY', 'CLUSTERS_EFEDS_MULTIPLE' etc
- Unit: N/A
- Data Type: String


#### xmatch_version

- Column Name: xmatch_version
- Description: The cross-match software version and OIR catalog used e.g. 'LSdr8-JWMS_v2.1LSdr8-JWMS_v2.1', 'LSdr8-AG_v1,JC_16032020', 'eromapper_2020-10-12', 'CW20ps1dr2-v011220' etc
- Unit: N/A
- Data Type: String


#### xmatch_dist

- Column Name: xmatch_dist
- Description: Distance between X-ray position and optical counterpart
- Unit: arcsec
- Data Type: Float32


#### xmatch_metric

- Column Name: xmatch_metric
- Description: Metric giving quality of cross-match. Meaning is dependent on xmatch_method, e.g. p_any for NWAY
- Unit: N/A
- Data Type: Float32


#### xmatch_flags

- Column Name: xmatch_flags
- Description: Flags indicating cross-match properties (e.g. status flags), xmatch_method dependent
- Unit: N/A
- Data Type: Int64


#### target_class

- Column Name: target_class
- Description: Best guess of source classification at time of xmatch e.g. 'GALAXY','STAR','QSO','UNKNOWN',....
- Unit: N/A
- Data Type: String


#### target_priority

- Column Name: target_priority
- Description: Relative to other targets in this catalog, interpreted/adapted later to derive a final target priority
- Unit: N/A
- Data Type: Int32


#### target_has_spec

- Column Name: target_has_spec
- Description: Flags used to indicate if target has good quality archival spectroscopy available
- Unit: N/A
- Data Type: Int64


#### opt_cat

- Column Name: opt_cat
- Description: Describes which OIR survey provided the optical counterpart for this row of the table, i.e. which OIR cat gives the entries in fields opt_ra, opt_dec, opt_pmra, opt_pmdec, opt_epoch, and which OIR identifier is given in the *_id columns
- Unit: N/A
- Data Type: String


#### ls_id

- Column Name: ls_id
- Description: Identifier of counterpart (if any) in dr20_legacy_survey_dr8 ('ls_id' field). Arithmetically derived from legacysurvey sweep file columns: release, brickid and objid:  ls_id = objid + (brickid * 2**16) + (release * 2**40)
- Unit: N/A
- Data Type: Int64


#### gaia_dr3_source_id

- Column Name: gaia_dr3_source_id
- Description: Identifier of counterpart (if any) in dr20_gaia_dr3_source ('source_id' field)
- Unit: N/A
- Data Type: Int64


#### opt_ra

- Column Name: opt_ra
- Description: Sky coordinate of optical/IR counterpart, included for validity checks only
- Unit: deg
- Data Type: Float64


#### opt_dec

- Column Name: opt_dec
- Description: Sky coordinate of optical/IR counterpart, included for validity checks only
- Unit: deg
- Data Type: Float64


#### opt_pmra

- Column Name: opt_pmra
- Description: Proper motion of optical/IR counterpart, included for validity checks only
- Unit: mas/yr
- Data Type: Float32


#### opt_pmdec

- Column Name: opt_pmdec
- Description: Proper motion of optical/IR counterpart, included for validity checks only
- Unit: mas/yr
- Data Type: Float32


#### opt_epoch

- Column Name: opt_epoch
- Description: Epoch of opt_ra,opt_dec
- Unit: decimal year
- Data Type: Float32


#### pkey

- Column Name: pkey
- Description: primary key of table entry
- Unit: N/A
- Data Type: Int64
