## Basic Information
Legacy Survey DR8 catalogue derived from 'sweep' catalogues.

For more information see <a href="<a href="https://www.legacysurvey.org/dr8/">https://www.legacysurvey.org/dr8/</a>">https://www.legacysurvey.org/dr8/">https://www.legacysurvey.org/dr8/</a></a>.

### Naming Convention
$MOS_TARGET/[V_TARG]/[FTYPE]/mos_legacy_survey_dr8.[FTYPE] with FTYPE='parquet' and V_TARG='2.0.0' for DR20.

## Example DATAFRAME List

### Columns


#### release

- Column Name: release
- Description: Unique integer denoting the camera and filter set used (RELEASE documentation: https://www.legacysurvey.org/release/)
- Unit: N/A
- Data Type: Int32


#### brickid

- Column Name: brickid
- Description: A unique Brick ID (in the range 1-662174)
- Unit: N/A
- Data Type: Int64


#### brickname

- Column Name: brickname
- Description: Name of brick, encoding the brick sky position, eg "1126p222" near RA=112.6, Dec=+22.2
- Unit: N/A
- Data Type: String


#### objid

- Column Name: objid
- Description: Catalog object number within this brick; a unique identifier hash is RELEASE,BRICKID,OBJID; OBJID spans 0 to N-1 and is contiguously enumerated within each blob
- Unit: N/A
- Data Type: Int64


#### type

- Column Name: type
- Description: Morphological model: "PSF"=stellar, "REX"="round exponential galaxy" = round EXP galaxy with a variable radius, "EXP"=exponential, "DEV"=deVauc, "COMP"=composite, "DUP"==Gaia source fit by different model. Note that in some FITS readers, a trailing space may be appended for "PSF ", "EXP " and "DEV " since the column data type is a 4-character string
- Unit: N/A
- Data Type: String


#### ra

- Column Name: ra
- Description: Right ascension at equinox J2000
- Unit: deg
- Data Type: Float64


#### dec

- Column Name: dec
- Description: Declination at equinox J2000
- Unit: deg
- Data Type: Float64


#### ra_ivar

- Column Name: ra_ivar

- Unit: 1/deg^2
- Data Type: Float32
