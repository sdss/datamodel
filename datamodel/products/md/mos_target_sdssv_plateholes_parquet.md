## Basic Information
The combination of the dr20_sdssv_plateholes and dr20_sdssv_plateholes_meta tables are a database representation of the SDSS-V platelist data product (<a href="https://svn.sdss.org/public/data/sdss/platelist/trunk/)">https://svn.sdss.org/public/data/sdss/platelist/trunk/)</a>.

These tables were used within early iterations of FPS target_selection as a way to communicate information about which targets had been included in SDSS-V plates. This information was used to e.g. de-prioritise targets that were expected to have a good quality spectroscopic measurement by the end of the SDSS-V plate observations.

The dr20_sdssv_plateholes_meta table contains meta-data for each SDSS-V plate (one record per plate), whereas the dr20_sdssv_plateholes table has one entry for each drilled hole in each SDSS-V plate. The dr20_sdssv_plateholes and dr20_sdssv_plateholes_meta tables should be joined via the yanny_uid field.

Mostly derived from: <a href="https://data.sdss.org/datamodel/files/PLATELIST_DIR/designs/DESIGNID6XX/DESIGNID6/plateDesign.html">https://data.sdss.org/datamodel/files/PLATELIST_DIR/designs/DESIGNID6XX/DESIGNID6/plateDesign.html</a>

### Naming Convention
$MOS_TARGET/[V_TARG]/[FTYPE]/mos_sdssv_plateholes.[FTYPE] with FTYPE='parquet' and V_TARG='2.0.0' for DR20.

## Example DATAFRAME List

### Columns


#### holetype

- Column Name: holetype
- Description: type of hole to be drilled; one of SDSS, BOSS, MARVELS, APOGEE, GUIDE, CENTER, TRAP, or ALIGNMENT; this really refers to the instrument to be used, so for example SEGUE targets should be either SDSS or BOSS
- Unit: N/A
- Data Type: String


#### targettype

- Column Name: targettype
- Description: type of target; one of SCIENCE, STANDARD, or SKY (if holetype is BOSS, SDSS, MARVELS or APOGEE); NA if not holetype is not one of the instruments (GUIDE, CENTER, TRAP, ALIGNMENT)
- Unit: N/A
- Data Type: String


#### sourcetype

- Column Name: sourcetype
- Description: indicate the nature of the source, one of STAR, QSO, GALAXY, or NA (if targettype is NA)
- Unit: N/A
- Data Type: String


#### target_ra

- Column Name: target_ra
- Description: right ascension, in J2000 deg
- Unit: N/A
- Data Type: Float64


#### target_dec

- Column Name: target_dec
- Description: declination, in J2000 deg
- Unit: N/A
- Data Type: Float64


#### iplateinput

- Column Name: iplateinput
- Description: integer indicating which plateInput file from the list in the plateDefinition file this came from (1-indexed); -1 or 0 if the hole wasn't from a plateInput file.
- Unit: N/A
- Data Type: Int32


#### pointing

- Column Name: pointing
- Description: which pointing (1-indexed) this target is associated with; note this means which pointing among the ones that this plate is designed for, so if this a single pointing plate but uses the B-side MARVELS fibers, this will be "1", not "2"
- Unit: N/A
- Data Type: Int32


#### offset_integer

- Column Name: offset_integer
- Description: which offset this target is associated with; 0 means it is not associated with an offset, it is associated with the primary pointing
- Unit: N/A
- Data Type: Int32


#### fiberid

- Column Name: fiberid
- Description: what fiber on the appropriate instrument (as listed in holetype) is assigned to this target (-9999 for none)
- Unit: N/A
- Data Type: Int32


#### block

- Column Name: block
- Description: which fiber block that fiber is associated with (-9999 for none)
- Unit: N/A
- Data Type: Int32


#### iguide

- Column Name: iguide
- Description: for guide holes, which guide fiber it is assigned to (from 1 to 11) (-9999 for non guide holes)
- Unit: N/A
- Data Type: Int32


#### xf_default

- Column Name: xf_default
- Description: X position of hole in focal plane, given a default set of observing parameters (hour angle of zero, temperature of 5 deg C), in units of mm; for position angle of zero (which we always use), +X is +RA
- Unit: N/A
- Data Type: Float32


#### yf_default

- Column Name: yf_default
- Description: Y position of hole in focal plane, given a default set of observing parameters (hour angle of zero, temperature of 5 deg C), in units of mm; for position angle of zero (which we always use), +Y is +Dec
- Unit: N/A
- Data Type: Float32


#### lambda_eff

- Column Name: lambda_eff
- Description: wavelength to optimize hole location for, in Angstroms (platedesign will default to 5400 if not set)
- Unit: N/A
- Data Type: Float32


#### zoffset

- Column Name: zoffset
- Description: backstopping offset distance, in microns (platedesign will default to 0 if not set)
- Unit: N/A
- Data Type: Float32


#### bluefiber

- Column Name: bluefiber
- Description: 1 if the BOSS instrument is meant to assign this target a "blue" fiber; 0 otherwise
- Unit: N/A
- Data Type: Int32


#### chunk
