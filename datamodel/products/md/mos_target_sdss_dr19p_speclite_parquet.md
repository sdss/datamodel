## Example DATAFRAME List

### Columns


#### plate

- Column Name: plate
- Description: Plate number (each plate corresponds to an actual plug plate)
- Unit: N/A
- Data Type: Int32


#### mjd

- Column Name: mjd
- Description: Modified Julian Day of observation
- Unit: day
- Data Type: Int32


#### fiberid

- Column Name: fiberid
- Description: Fiber number
- Unit: N/A
- Data Type: Int32


#### field

- Column Name: field
- Description: Field identifier
- Unit: N/A
- Data Type: Int32


#### catalogid

- Column Name: catalogid
- Description: The associated ID in the dr20_catalog table
- Unit: N/A
- Data Type: Int64


#### bestobjid

- Column Name: bestobjid
- Description: Unique database ID of (recommended) position-based photometric match based on RUN, RERUN, CAMCOl, FIELD, ID (same as SkyServer version)
- Unit: N/A
- Data Type: Int64


#### targetobjid

- Column Name: targetobjid
- Description: Unique database ID of targeting object based on RUN, RERUN, CAMCOl, FIELD, ID (same as SkyServer version)
- Unit: N/A
- Data Type: Int64


#### run2d

- Column Name: run2d
- Description: Spectroscopic 2D reduction (extraction of spectra) name
- Unit: N/A
- Data Type: String


#### run1d

- Column Name: run1d
- Description: Spectroscopic 1D reduction (redshift and classification) name
- Unit: N/A
- Data Type: String


#### firstrelease

- Column Name: firstrelease
- Description: Name of first release this PLATE, MJD, FIBERID, RUN2D was associated with
- Unit: N/A
- Data Type: String


#### plug_ra

- Column Name: plug_ra
- Description: Right ascension of hole (J2000)
- Unit: degrees
- Data Type: Float64


#### plug_dec

- Column Name: plug_dec
- Description: Declination of hole (J2000)
- Unit: degrees
- Data Type: Float64


#### sourcetype

- Column Name: sourcetype
- Description: String expressing type of source (similar to OBJTYPE in DR8 and earlier)
- Unit: N/A
- Data Type: String


#### objtype

- Column Name: objtype
- Description: Why this object was targeted. Note that if this field says QSO, it could be the case that this object would have been targetted as a GALAXY or any number of other categories as well. The PRIMTARGET and SECTARGET flags in the plug-map structure (in the spField file) gives this full information.
- Unit: N/A
- Data Type: String


#### targettype

- Column Name: targettype
- Description: General type of target ("SCIENCE", "STANDARD" or "SKY")
- Unit: N/A
- Data Type: String


#### lambda_eff

- Column Name: lambda_eff
- Description: Effective wavelength drilling position was optimized for
- Unit: Angstroms
- Data Type: Float32


#### sn_median_all

- Column Name: sn_median_all
- Description: Median signal-to-noise per pixel across full spectrum (aka SN_MEDIAN_ALL)
- Unit: N/A
- Data Type: Float32


#### sn_median_u

- Column Name: sn_median_u
- Description: Median signal-to-noise per pixel within the u bandpass
- Unit: N/A
- Data Type: Float32


#### sn_median_g

- Column Name: sn_median_g
- Description: Median signal-to-noise per pixel within the g bandpass
- Unit: N/A
- Data Type: Float32


#### sn_median_r

- Column Name: sn_median_r
- Description: Median signal-to-noise per pixel within the r bandpass
- Unit: N/A
- Data Type: Float32


#### sn_median_i

- Column Name: sn_median_i
- Description: Median signal-to-noise per pixel within the i bandpass
- Unit: N/A
- Data Type: Float32


#### sn_median_z

- Column Name: sn_median_z
- Description: Median signal-to-noise per pixel within the z bandpass
- Unit: N/A
- Data Type: Float32


#### platesn2

- Column Name: platesn2
- Description: Overall signal-to-noise-squared measure for plate (only for SDSS spectrograph plates)
- Unit: N/A
- Data Type: Float32


#### platequality

- Column Name: platequality
- Description: Characterization of plate quality.
- Unit: N/A
- Data Type: String


#### z

- Column Name: z
- Description: Best redshift
- Unit: N/A
- Data Type: Float32


#### z_err

- Column Name: z_err
- Description: Error in best redshift
- Unit: N/A
- Data Type: Float32


#### zwarning

- Column Name: zwarning
- Description: Bitmask of spectroscopic warning values; 0 means everything is OK
- Unit: N/A
- Data Type: Int32


#### zwarning_noqso

- Column Name: zwarning_noqso
- Description: Best redshift when ignoring QSO fits, recommended for BOSS CMASS and LOWZ targets; calculated only for survey='boss' spectra, not for any SDSS spectrograph data
- Unit: N/A
- Data Type: Int32


#### class

- Column Name: class
- Description: Best spectroscopic classification ("STAR", "GALAXY" or "QSO")
- Unit: N/A
- Data Type: String


#### subclass

- Column Name: subclass
- Description: Best spectroscopic subclassification
- Unit: N/A
- Data Type: String


#### tfile

- Column Name: tfile
- Description: File that best fit template comes from in idlspec1d product
- Unit: N/A
- Data Type: String


#### specprimary

- Column Name: specprimary
- Description: Objects observed multiple times will have this set to 1 for one observation only. This is usually the 'best' observation.
- Unit: N/A
- Data Type: Int32


#### nspecobs

- Column Name: nspecobs
- Description: Number of spectral observations.
- Unit: N/A
- Data Type: Int32


#### pk

- Column Name: pk
- Description: A unique identifier for this table row.
- Unit: N/A
- Data Type: Int64
